# General Code Quality Instructions

- Generate clean, readable, maintainable code.
- Prefer clarity over cleverness; avoid unnecessary complexity.
- Use consistent, descriptive names for files, variables, and functions.
- Write modular and reusable functions; avoid duplication.
- Handle all errors, edge cases, and undefined/null states.
- Use early returns to reduce indentation and improve readability.
- Follow SOLID principles when appropriate.
- Avoid side effects and prefer pure functions where possible.
- Document non-obvious logic with short, clear comments.
- Optimize performance without sacrificing clarity.
- Use Prettier and standard linting conventions.
